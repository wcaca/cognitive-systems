# Research Map · 研究地图

> 整个仓在研究什么、各方向之间的关系、为什么这么分。

## 核心问题

**怎么构建一个完整的、严密的、适应性强的、可恢复性强的认知系统？**

"认知系统"在本文里指：能感知、能推理、能行动、能学习、能恢复的主体或主体集合。包括但不限于：
- agent（单个或多个）
- 大语言模型及其组合
- 可能的意识系统
- 人机混合系统

## 三大研究方向

### 1. Agent（运行系统）
研究"让 agent 跑起来的系统"该怎么设计。包括：
- agent 与 harness 的关系
- 失败模式、恢复机制
- 调度、记忆、工具、权限
- 多 agent 协作与对抗

**首个落地**：[`20-systems/agent-harness/`](../../20-systems/agent-harness/)

### 2. LLM（大模型）
研究大语言模型本身的能力边界、对齐问题、评测方法。包括：
- 能力图谱（什么能做、什么不能做）
- 评测方法（怎么测才准）
- 对齐与安全
- 训练-推理闭环

**计划**：[`20-systems/llm-frontier/`](../../20-systems/llm-frontier/)

### 3. Consciousness（意识）
研究意识本身：它是什么、机器能否有意识、怎么验证。包括：
- 意识的理论框架（IIT、GWT、Global Workspace 等）
- 可观测的代理指标
- 哲学与科学的边界

**计划**：[`20-systems/consciousness/`](../../20-systems/consciousness/)

## 三者的关系

```
        ┌──────────────────────┐
        │   Cognitive System   │
        └──────────┬───────────┘
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
    ┌──────┐   ┌──────┐   ┌──────────┐
    │ Agent│   │ LLM  │   │Conscious-│
    │      │   │      │   │  ness    │
    └──┬───┘   └──┬───┘   └────┬─────┘
       │          │            │
       └──────────┴────────────┘
                  │
         交叉：可恢复性、协议、诚实
```

- **Agent** 是"系统怎么组织"的问题
- **LLM** 是"底层能力从哪来"的问题
- **Consciousness** 是"系统在感受什么"的问题

三者在工程上耦合（LLM 跑在 agent 系统里，agent 系统可能有意识或没意识），在哲学上纠缠。

## 我们的研究方法

1. **从工程问题出发**：先有可运行的系统，再抽象理论
2. **失败驱动**：每个新理论必须能解释某类失败
3. **可证伪优先**：所有声称都给出"什么情况下这个声称是错的"
4. **跨方向对比**：agent 研究的发现会反馈给 LLM 研究，LLM 边界会反馈给意识研究

## 不研究的

- 单纯的语言学/语法问题
- 商业模型/产品策略
- 短期的工程 hack（除非上升为通用模式）
