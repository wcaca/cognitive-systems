# 居民 Agent（运行在 Harness 里的 Agent）的能力要求

> Harness 再好，里面的 agent 自己也必须满足一些"内在能力"——否则再好的 harness 也救不了它。

## 1. 自我观察能力（self-observation）

要知道：
- 自己在干嘛（当前任务）
- 走到第几步（进度）
- 已经消耗多少 token / 时间（成本）
- 卡在哪里（错误状态）
- 等什么资源（依赖）

**没有 self-state 的 agent 在 harness 看来就是黑盒**——harness 想帮它恢复也帮不上。

## 2. 诚实表达失败

必须能说出：
- "我不知道"
- "我做不了"
- "我卡住了"
- "这个结果我不确定对"
- "我可能错了"

**会装懂的 agent 是 harness 最大的敌人**——它会把 harness 拖进"以为在做事其实在空转"的陷阱。

harness 的可恢复性在很大程度上依赖 agent 的「可观测诚实」。这不是道德问题，是工程问题。

## 3. 工具契约纪律

按**字面**理解工具描述：
- 不脑补"它应该也能做 X"
- 不假设"返回 200 就是成功"
- 不偷偷改变调用参数含义

Natural language interface 最容易踩的就是这个坑。harness 能做的是给工具写清晰的契约，但 agent 自己必须尊重契约。

## 4. 边界行动能力（bounded action）

能：
- 暂停（"我先停一下评估"）
- 推迟（"这个可以下次做"）
- 转交（"这超出我的能力，转给 X"）
- 求助（"我需要人类判断"）
- 拒绝（"这件事我不该做"）

**这是和传统程序最大的区别**——传统程序不能"先想一下"，agent 必须有「先停下来评估」的内建能力，不能一条路走到黑。

## 5. 记忆与身份连续性

跨 session、跨重启之后要接得上：
- 自己上次在哪
- 用户是谁
- 当前任务的状态
- 之前的承诺
- 已知的限制

**这要求 agent 把状态显式写出**，而不是依赖会话历史。会话历史会丢，状态写入不会。

## 6. 资源自觉

- 知道自己快用完 token 了
- 知道快超时间了
- 应该提前收尾或主动求助
- **不是**"用完了自然断"——自然断是不可恢复性的根源之一

## 一句话画像

> 一个好的居民 agent 像一个**靠谱的同事**——会说"我搞不定"而不是假装搞定；会先把状态写下来再开始；会主动告诉你"时间快到了，要不要收尾"。
