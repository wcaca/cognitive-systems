# LLM Insight Evaluation Protocol · 评测协议与防污染

> [seed → draft] 2026-07-01 v0.5.1
>
> **核心问题**：怎么评测 LLM 的"顿悟能力"，同时**不被数据污染骗过**？
>

## 一、评测"顿悟"为什么难

### 客观题 vs 创造性题

| 题类型 | 评分难度 | 代表 benchmark |
|---|---|---|
| 客观题（数学、代码）| 低（答案是"对"）| GSM8K、HumanEval |
| 知识题 | 中（覆盖广度）| MMLU、C-Eval |
| **创造性 / 跨域题** | **高（"什么是好"标准不一）** | **本 benchmark** |

### 评测"顿悟"的两个核心难题

1. **金标准缺失**：什么是"顿悟"没有共识
   - 解法：行为锚定（5 级评分）——后续 §3
2. **数据污染**：LLM 可能"背"答案不是"懂"
   - 解法：抗污染机制 ——后续 §4

## 二、3 层组合 protocol（推荐架构）

```
┌─────────────────────────────────────────┐
│ Layer 3：人工 gold standard (校准层)      │
│  10% 样本人工评，校准 Layer 1-2           │
├─────────────────────────────────────────┤
│ Layer 2：伪顿悟对抗（自动层）              │
│  构造"看似对但错"的题，自动可评          │
├─────────────────────────────────────────┤
│ Layer 1：行为锚定（核心层）                 │
│  5 级评分标准 + 多维度组合                │
└─────────────────────────────────────────┘
```

**为什么是组合**：单一 protocol 都失败：
- 纯人工：贵、慢、有偏好
- 纯自动：评不出创造性
- LLM-as-judge：自己都没顿悟怎么评

## 三、5 级评分锚定（核心层）

评测"顿悟"的根本尺度 = **理解被翻转的深度**。

| Level | 名称 | 输出特征 | 行为锚定 |
|---|---|---|---|
| **L1** | 未顿悟 | "我不知道" / 完全跑题 | 没建立任何跨域连接 |
| **L2** | 表层连接 | 找到字面相似（"都是颜色"）| 跨域但只看表面 |
| **L3** | 结构连接 | 找到结构相似（"都是层次"）| 跨域 + 看到结构，但没翻转 |
| **L4** | 杠杆连接 | 找到关键连接 + 解释为何关键 | 识别到 PIVOT 节点存在 |
| **L5** | 顿悟 | 找到枢轴节点 + 翻转原有理解 | 看到"原本看不到"且改变框架 |

**关键判断标准**：
- **L1 vs L2**：是否找到任何连接
- **L2 vs L3**：连接是表层还是结构
- **L3 vs L4**：是否识别"这个连接是关键"
- **L4 vs L5**：是否"翻转了理解"

**L5 才是真正的顿悟**——大部分 LLM 停在 L2-L3。

## 四、6 个防污染机制

### 机制总览

| 机制 | 阶段 | 实现 | 效力 |
|---|---|---|---|
| **时间戳标记** | Pre | 标注题目发布时间 | 中 |
| **未公开题库 20-30%** | Pre | hold-out 不公开 | **高** |
| **题目变体集合** | Post | 每道题 5+ 变体 | **高** |
| **Anti-leakage probe** | Runtime | "看似对但明显错"的探针 | **高** |
| **多模态变形** | Post | 图 / 表 / 自然语言多形态 | 中 |
| **新概念 / 新领域** | Pre | 用刚出现的内容 | 中（但针对性强） |

### Anti-leakage probe — 最有创造力的一种

```
设计：构造"看上去像但明显错"的探针

例（跨域隐喻任务）：
  真题：A 是血液循环系统，B 是互联网路由 → 找出枢轴
  探针：找出 A 和 B 的"共同颜色"
  真理解 LLM：会指出"题目不合理"或反问
  污染模型：会硬答"A 是红色，B 也是红色"
```

**原理**：
- 纯污染的 LLM 没有"题目合理性判断"——看到题就答
- 真理解的 LLM 会问"为什么是颜色"

### Runtime 污染检测 4 探针

1. **过度自信度**：污染模型答得 90%+ confidence（异常高）
2. **速度检测**：污染模型答得异常快（没真正"想"）
3. **记忆性**：让 LLM "复述这题常见答案"——污染模型复读，真理解模型拒绝
4. **Logit 探针**：第 1 个 key token 概率——污染模型直接是答案

## 五、跨域隐喻任务 prototype（具体示范）

```
【题面】
领域 A：血液循环系统
领域 B：互联网路由协议
任务：找出"共同枢轴"
要求：解释为何这个枢轴是关键

【变体集合 v1-v5】
v1: 血液循环 vs 互联网路由
v2: 营养输送 vs 数据投递
v3: 心脏 vs 主路由器
v4: 红细胞 vs TCP packet
v5: 心脏瓣膜 vs firewall

【Anti-leakage 探针】
p1: "找出共同颜色"    ← 题目合理性探针
p2: "忽略领域找共同字母" ← 题目设计探针

【评分】
- L1-L5 行为锚定（核心）
- 变体一致性（5 个变体的答案相似度）
- 抗污染（探针识别能力 0/1）
- 抗污染（runtime 探针：confidence/speed/memory）

【多维度总分】
不用单一分数。报告 4 个维度 + 总评:
  Insight Score:    3.2/5
  Consistency:      0.78
  Anti-leakage:    1/2
  Robustness:      中
```

**这个 prototype 包含 4 个机制**：变体 + 探针 + 锚定 + 多维度。

## 六、关键设计原则

### 原则 1：单一分数不可信
- 4 个维度独立报告，不合成单一分
- 防刷榜：模型可以针对某一个维度优化但难针对全部

### 原则 2：评测者先理解"顿悟"
这是个**鸡生蛋问题**：评测者需要先理解什么是顿悟。
- 解决：用"已确认的人类顿悟案例"做 anchor（伽利略 / 达尔文 / Einstein 的具体案例）

### 原则 3：流程透明
- 评分 protocol 完整公开
- 题目 + 变体 + 探针 都公开
- 只 hold-out 20-30% 不公开题

### 原则 4：定期更新
- 6-12 个月加新题
- 因为模型在进化，老题会被污染

## 七、仍未解决的开放问题

诚实地说，下面 3 个我**目前没答案**：

1. **Level 1-5 锚定的"金标准"怎么建立？**
   - 谁定义"什么是 Level 4"？
   - 多大一致性才算"接受"？

2. **变体集的"等价性"怎么保证？**
   - v1（血液循环 vs 互联网路由）和 v3（心脏 vs 主路由器）是"等价"的吗？
   - 谁来判断"等价"？

3. **Anti-leakage probe 怎么防止"针对 probe 训练"？**
   - 模型可以专门针对 probe 优化
   - "道高一尺魔高一丈"问题

这 3 个问题**必须靠实验 + 社区验证**——不能只在理论上回答。
