# Insight Completion Rubric · 完成度评分 Rubric

> [draft] 2026-07-01 v0.6.4
>
> 配套 [`80-meta/insight-to-work.md`](./insight-to-work.md) 和 [`insight-4-persistence-mechanisms.md`](./insight-4-persistence-mechanisms.md) 的"多维评分"使用。
>
> **核心目的**：把"什么时候算完成"从抽象判断变成**可重复、可校准、可审计**的具体规则。

---

## 一、为什么需要 rubric

LLM 的"完成了"判断有问题：
- **过早停**：写了几句就以为完成
- **过晚停**：完美主义一直补细节
- **伪装停**：给一个看起来"完成"的版本但实际缺口很多

**rubric = 把"完成"从主观判断变成显式规则**。

## 二、评分维度的 3 层 × 3 子分 = 9 维

### 3 层（来自 v0.6.2）

| 层 | 内容 | 满分 |
|---|---|---|
| Layer 1 | insight 核心 | 10 |
| Layer 2 | 4 面展开 | 4 × 10 = 40 |
| Layer 3 | 制品本身 | N × 10（N = 制品数） |

### 3 子分（每个维度都拆）

每个层的每个分数 = 3 个子分数的平均：

| 子分 | 含义 | 权重 |
|---|---|---|
| **完整性** | 最低门槛全达到？ | 33% |
| **准确性** | 内容真实 / 正确？ | 33% |
| **可应用性** | 换场景能用？ | 33% |

### 总分公式

```
总分 = Layer1 × 0.2 + Layer2_avg × 0.5 + Layer3_avg × 0.3
     = 0.2 × 10 + 0.5 × 10 + 0.3 × 10 = 10
```

（按 v0.6.3 的设计：Layer 2 最重 = 50%，Layer 1 和 Layer 3 各 20-30%）

## 三、Layer 1 评分标准（insight 核心）

### 10 分拆解

```
9-10: 能在 30 秒内说清，关键性"翻转型"清晰
7-8:  能说清但关键性不够"翻转"
5-6:  能说清但关键性只是"相关"
3-4:  能说清但关键性缺失
1-2:  不能在 30 秒内说清
0:    缺失
```

### 3 子分评分

| 子分 | 9-10 标准 |
|---|---|
| 完整性 | 1 句话 + 关键性声明，2 个字段都在 |
| 准确性 | 没事实错误 / 没逻辑跳跃 |
| 可应用性 | 换 2-3 个不同领域都能复用 |

## 四、Layer 2 评分标准（4 面展开）

### 每面 10 分拆解

```
9-10: 所有最低门槛达标 + 内容准确 + 有反例
7-8:  最低门槛达标，准确但缺反例
5-6:  最低门槛达标，但"假完成"
3-4:  部分门槛达到
1-2:  多数门槛缺失
0:    完全没展开
```

### 每面 3 子分评分

| 子分 | 9-10 标准 |
|---|---|
| 完整性 | 该面最低门槛**全部**达标（参考 v0.6.1）|
| 准确性 | 内容真实，无虚构 / 循环引用 |
| 可应用性 | 至少 1 个具体操作步骤可执行 |

### 4 面的差异化权重

| 面 | 权重 | 为什么 |
|---|---|---|
| 历史面 | 0.20 | 可有可无，不达标可降级 |
| 结构面 | 0.30 | 必须有，否则理解不深 |
| 应用面 | 0.30 | 必须有，否则"自嗨" |
| 限制面 | 0.20 | 必须有，但可简略 |

**Layer 2 总分 = 加权平均**。

## 五、Layer 3 评分标准（每个制品）

### 制品分类与对应标准

| 类型 | 完整性 | 准确性 | 可应用性 |
|---|---|---|---|
| **doc**（文档）| 每节齐全 + 链接清晰 | 事实正确 | 读者能 follow |
| **code**（代码）| 所有函数有注释 + 边界覆盖 | 测试通过 | 能在新环境跑 |
| **image**（图）| 标注 + 图例 + 来源 | 事实对应 | 简化版 / 详细版都有 |
| **model**（模型）| 输入输出 schema 完整 | 验证集通过 | 换数据可用 |
| **protocol**（协议）| 步骤 + 边界 + 反例 | 真实可执行 | 换工具可移植 |

### 每个制品的 10 分拆解

```
9-10: 完整 + 准确 + 可应用三层都 9 分+
7-8:  完整 + 准确，但应用层只到示例
5-6:  完整但准确性可疑
3-4:  部分完整
1-2:  半成品
0:    缺失
```

### Layer 3 总分 = 多个制品的平均

```
Layer_3_avg = avg(artifact_1_score, artifact_2_score, ...)
```

如果某些制品是关键（其他依赖它们），可**加权**：
```
Layer_3 = sum(artifact_weight × artifact_score) / sum(artifact_weight)
```

## 六、3 类应用的完成阈值

按 v0.6.3 的设计，**阈值因应用场景而异**：

### 场景 A：科研 / 工程（高严谨）

```
完成条件:
  total ≥ 8.5
  Layer 1 ≥ 9.0
  Layer 2 每面 ≥ 8.0
  Layer 3 每个制品 ≥ 8.0

不达标 → 不算完成，继续补
```

### 场景 B：快速 prototype

```
完成条件:
  total ≥ 6.5
  Layer 1 ≥ 7.0
  Layer 2 每面 ≥ 6.0
  Layer 3 每个制品 ≥ 5.0

不达标 → 列出待办，不一定现在补
```

### 场景 C：公开发布

```
完成条件:
  total ≥ 9.0
  Layer 1 ≥ 9.5
  Layer 2 每面 ≥ 9.0
  Layer 3 每个制品 ≥ 9.0

不达标 → 不发布
```

## 七、评分的工作流

### 评分时机

- **每个 checkpoint**：自动评分
- **每个 Step 完成**：手动评分（更严）
- **最终验收**：评分 + 解释 + 复审

### 评分流程

```
Step 1: 收集证据
  - Layer 1 文件
  - Layer 2 checklist 状态
  - Layer 3 制品 + 元数据

Step 2: 9 维打分（每维 0-10）
  - L1 完整性 / 准确性 / 可应用性
  - L2 4 面 各自 3 子分
  - L3 每个制品 各自 3 子分

Step 3: 计算总分
  - 应用对应场景的完成阈值
  - 通过 → 完成
  - 不通过 → 列出缺口 + 优先级

Step 4: 写评分解释
  - 为什么这个分数
  - 缺什么
  - 下一步建议
```

### 解释模板（强制）

```markdown
## 评分 · [timestamp]

### 总分：[X.X / 10]

### Layer 1：[X/10]
- 完整性：[Y/10] + 1 句话理由
- 准确性：[Y/10] + 1 句话理由
- 可应用性：[Y/10] + 1 句话理由

### Layer 2：[X/10]
- 历史面：[Y/10]（完整性/准确性/可应用性）
- 结构面：[Y/10]
- 应用面：[Y/10]
- 限制面：[Y/10]

### Layer 3：[X/10]
- 制品 1（[type]）：[Y/10]
- 制品 2（[type]）：[Y/10]
- ...

### 应用场景：[场景 A/B/C]
### 阈值：[X.X]
### 通过：✓ / ⚠ / ✗

### 缺口清单
- [缺口 1 + 优先级]
- [缺口 2 + 优先级]
- ...

### 下一步
- [具体动作]
```

## 八、rubric 的反模式

### 反模式 1：凑分数
- LLM 只为了"分数 ≥ 阈值"凑字数
- **检测**：每面必须有"具体证据"链接，否则该子分 0

### 反模式 2：评分僵化
- 阈值固定，不因场景调整
- **检测**：场景必须显式标注

### 反模式 3：评分变成装饰
- 评分不进入实际决策
- **检测**：评分必须挂"动作"（通过 / 不通过 / 待办）

### 反模式 4：评分者单一
- 永远 LLM 自己评
- **检测**：评分必须有 peer review（人工或另一个 LLM）

## 九、rubric 在 mavis harness 的应用

### 评分脚本（建议）

```bash
#!/bin/bash
# scripts/compute-score.sh
# 给定 insight 路径，输出评分

INSIGHT=$1
SCENARIO=${2:-research}  # 默认科研场景

# 检查 Layer 1
LAYER1=$(yq '.insight_core | length' "$INSIGHT/INSIGHT-NAME-core.md")

# 检查 Layer 2
for face in 历史面 结构面 应用面 限制面; do
  SCORE=$(yq ".faces_expansion.$face | length" "$INSIGHT/INSIGHT-NAME-faces.md")
  # ...
done

# 计算总分
TOTAL=$(...)
echo "总分: $TOTAL / 10"
echo "场景: $SCENARIO"
echo "通过: ..."
```

### 集成到 v0.4 决策流程

- **PIVOT echo 后**：立即评 Layer 1
- **4 面 checklist 后**：评 Layer 2
- **制品生成后**：评 Layer 3
- **完成时**：综合评分 + 决策（继续 / 完成 / 暂停）

## 十一、一句话总结

> **rubric = 把"完成"从主观判断变成 9 维 3 子分的可重复规则**。
> 
> **3 层 × 3 子分 = 9 维 × (完整性 + 准确性 + 可应用性) = 27 个具体评分点**。
> 
> **完成 = total ≥ 场景阈值 + 每面 ≥ 最低门槛 + 评分有解释有动作**。
> 
> **rubric 不被场景调整 = 评分僵化；rubric 不进入决策 = 评分装饰**。
